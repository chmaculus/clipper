!short:RAT_MENU2()          *
'------------------------------------------------------------------------------

 ^bRAT_MENU2()

 -[bMouse] is an optional codeblock for mouse clicks. If the
  mouse click does not satisfy RAT_MENU2(), and there is a
  [bMouse] codeblock, it will be evaluated as follows:
               eval(bMouse,mouserow, mousecol)
!seealso: 
'------------------------------------------------------------------------------



!short:RAT_READ()           *
'------------------------------------------------------------------------------

 ^bRAT_READ()

 RAT_READ() has been completely rewritten in 3.5, to more closely
 match GETSYS.PRG.

 -[bLmouse] (new in 3.5) is a codeblock that is evaluated when there
  is a mouse click and it is not meaningful to RAT_READ().
  It is evaluated as :  eval(bLMouse,mouserow, mousecolumn)
!seealso: 
'------------------------------------------------------------------------------



!short:SACHOICE()           *
'------------------------------------------------------------------------------

 ^bSACHOICE()

 -[nMRow, nMCol]  (new in 3.5) Directs sachoice() to draw
  the "[][]" for mouse up/down at nMrow, nMCol, and to be aware of mouse
  clicks on these buttons. (the screen under the arrows is saved/restored)

 -[bMouse] is a codeblock to evaluate mouse clicks other than those
  meaningful to sachoice(). The codeblock is evaluated as follows:
            eval(bMouse,mouserow, mousecolumn)
!seealso: 
'------------------------------------------------------------------------------

!short:^bChanges in SuperLib 3.0
!short:
!short:General changes
'------------------------------------------------------------------------------

 ^General

 (other than overall 5.01 optimization, conversion of privates and
  publics to locals and statics, use of hungarian notation, use
  of Tbrowse where indicated, use of codeblocks for speed... )

 - No direct screen writing - all goes through Clipper now

 - All internal procs and variables are now STATIC

 - In most cases, deleted records for internal files are recycled
   so that packing isn't needed.

!short:CLABEL()            *
'------------------------------------------------------------------------------

 ^bCLABEL()

 -This one was practically a rewrite

 -More/different menu structure

 -There are now switches for compressing blank lines and compressing
  blank spaces when printing. Previously blank lines were shifted out,
  and blank spaces were compressed to one.

 -Storage is now done to a DBF. CLABELS.DBF by default. See SLSF_*().

 -Expression builder is now an option

 -Editing label contents has been completely redone, and now had options
  for FIELD, TEXT, SPACES, EXPRESSION (optional). On character field, has
  options for TRIM(), UPPER(), LOWER(), PROPER().

 -Now supports Avery Laser labels directly

!short:TAGIT()             *
'------------------------------------------------------------------------------

 ^bTAGIT()

 -There are slight differences in behavior from previous versions.
  Previous versions required an array of fixed length, and only allowed
  tagging up to that length. This version grows or shrinks the array
  as needed. This version will also shrink an empty array down to
  zero on startup.

 -Allows TAG ALL MATCHING now after a search. Uses the #define MAXTAG
  which is at the top of the .PRG and which is set at 1000. You can
  change this for anything up to 4096 - max array elements.
!seealso:
