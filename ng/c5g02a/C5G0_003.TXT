!short: ^bOverview -- Runtime Errors^b                                                  ^r^CEA^r
'------------------------------------------------------------------------------
 ^bRuntime Errors^b

    This section is a summary of error messages that are possible when
    executing a Clipper application using the supplied subsystems.  The
    messages are divided into categories according to subsystem.  Each
    category is described below, followed by a listing of all messages in
    each category.

    ^bBASE Errors^b

    BASE error messages indicate errors generated by the Base system.  The
    general format of a BASE error message is as follows:

    ^bError | Warning BASE/xxxx <message text> <filename> | <operation>

    ^bTERM Errors^b

    TERM error messages indicate errors generated by the Terminal
    subsystem.  The general format of a TERM error message is as follows:

    ^bError | Warning TERM/xxxx <message text> <filename> | <operation>

    ^bDBFNTX/DBFNDX Errors^b

    DBFNTX/DBFNDX error messages indicate that an error occurred during a
    database or index operation utilizing either the DBFNTX or the DBFNDX
    database driver.  The general format of a DBFNTX/DBFNDX error message
    is as follows:

    ^bError | Warning DBFNTX/xxxx <message text> <filename> | <operation>
!seealso:

!short: BASE/1001    Undefined function                                             ^r^CEA^r
'------------------------------------------------------------------------------
 ^bBASE/1001   Undefined function

    ^bProblem:  ^bThe specified function or procedure was not linked with the
    application or was never directly referred to by the application.

    ^bSolution:  ^bMake sure the function is linked.  If the function is
    referred to only in an index key or other macro expression, add an
    EXTERN statement for the function.  For more information, refer to the
    Basic Concepts chapter in the Reference Book.
!seealso: 
'------------------------------------------------------------------------------



!short: BASE/1002    Undefined alias                                                ^r^CEA^r
'------------------------------------------------------------------------------
 ^bBASE/1002   Undefined alias

    ^bProblem:  ^bThe specified alias is not currently associated with any work
    area.

    ^bSolution:  ^bMake sure the appropriate database file is open.  Check to
    see if more than one work area uses the same alias at the same time.
    If so, designate a unique alias for each work area.  The SELECT()
    function can be used to see if an alias is currently associated with
    any work area.  For more information, refer to the Basic Concepts
    chapter in the Reference Book.
!seealso: 
'------------------------------------------------------------------------------



!short: BASE/1003    Undefined variable                                             ^r^CEA^r
'------------------------------------------------------------------------------
 ^bBASE/1003   Undefined variable

    ^bProblem:  ^bThe specified MEMVAR or FIELD variable does not exist or is
    not visible.

    ^bSolution:  ^bIf the variable is a database field, make sure the
    appropriate database file is open and that the appropriate work area is
    selected.  If the variable is a PRIVATE or PUBLIC variable, make sure
    the variable has been created via a PRIVATE or PUBLIC statement.  If
    the variable is a LOCAL or STATIC variable referred to via a macro,
    rearrange the code to refer to the variable directly (LOCAL and STATIC
    variables are not visible in macros).  The /W compiler option may be
    useful in finding misspelled or incorrectly specified variable
    references.  For more information, refer to the Basic Concepts chapter
    in the Reference Book.
!seealso: 
'------------------------------------------------------------------------------



!short: BASE/1004    No exported method                                             ^r^CEA^r
'------------------------------------------------------------------------------
 ^bBASE/1004   No exported method

    ^bProblem:  ^bA message was sent using the send operator (:), but the left
    operand is not an object or the object has no method with the specified
    name.  This error may also occur if the first parameter to the EVAL()
    function is not a code block or if a value other than a code block is
    supplied in a context where a code block is required.

    ^bSolution:  ^bCheck to make sure that the value being operated on is of
    the proper type or class.  If the error occurs on the send operator
    (:), make sure the message is one of the messages defined for the
    class.  If the missing method is "EVAL," make sure that code blocks
    have been supplied where required.  For more information, refer to the
    Basic Concepts and Standard Classes chapters in the Reference Book.
!seealso: 
'------------------------------------------------------------------------------



!short: BASE/1005    No exported variable                                           ^r^CEA^r
'------------------------------------------------------------------------------
 ^bBASE/1004   No exported variable

    ^bProblem:  ^bAn attempt was made to assign an exported instance variable
    using the send operator (:), but the left operand is not an object or
    the object has no exported variable with the specified name.

    ^bSolution:  ^bCheck to make sure that the value being operated on is of
    the proper type or class.  Make sure the name of the instance variable
    is spelled properly.  For more information, refer to the Basic Concepts
    and Standard Classes chapters in the Reference Book.
!seealso: 
'------------------------------------------------------------------------------
