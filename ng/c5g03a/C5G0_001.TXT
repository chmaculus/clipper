!short: _parc()     Retrieve a character parameter
'------------------------------------------------------------------------------
 ^b_parc()^b
 Retrieve a character parameter
ÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄ

 ^bC Prototype^b

     ^b#include "Extend.h"^b
     ^bchar *_parc(int paramNum [, int arrayIndex])^b

 ^bArguments^b

     ^BparamNum^B is the position of the parameter in the parameter list.

     ^BarrayIndex^B is an array index to specify a particular element if the
     indicated parameter is an array.

 ^bReturns^b

     _parc() returns a far pointer to a series of bytes representing the
     character value.

 ^bDescription^b

     _parc() is used to retrieve a character value passed as a parameter
     from Clipper.

     The pointer returned by _parc() may or may not point to the actual
     character value passed from Clipper.  In some cases, the pointer may
     point to a copy of the actual value.  In other cases, it may point to
     an internal value that is shared by more than one Clipper variable.

     The pointer returned from _parc() should not be used to directly modify
     the character value.  Doing so will produce unpredictable results.  To
     modify a character value, use the _retc() interface function to return
     a modified copy of the value, or pass the associated Clipper parameter
     by reference and use the _storc() interface function to modify it.

 ^bExamples^b

     From C:

     char *str;
     str = _parc(1);

     From Assembly language:

     EXTRN   __parc:FAR
       mov   ax, 1
       push  ax
       call  __parc         ; pointer returned in DX:AX
       add   sp, 2          ; reset stack pointer

 ^bFiles:^b  Library is CLIPPER.LIB, header file is Extend.h.
!seealso: "_parclen()" "_retc()" "_ret()" "_storc()" "_xunlock()"

 ^bC Prototype^b

     ^b#include "Extend.h"^b
     ^bdouble _parnd(int paramNum [, int arrayIndex])^b

 ^bArguments^b

     ^BparamNum^B is the position of the parameter in the parameter list.

     ^BarrayIndex^B is an array index to specify a particular element if the
     indicated parameter is an array.

 ^bReturns^b

 ^bDescription^b

     _parnd() is used to retrieve a numeric value passed as a parameter from
     Clipper.  The value is converted to a double precision floating point

     from C, the double value is automatically copied from the buffer into
     your C variable.  If you call from Assembly language, however, you
     should take care to preserve the double value, as subsequent calls to
     _parnd() will overwrite the buffer.

 ^bExamples^b

     From C:

     double num;
     num = _parnd(1);

     From Assembly language:

     EXTRN   __parnd:FAR
       mov   ax, 1
       push  ax
       call  __parnd        ; value pointed to by DX:AX
       add   sp, 2          ; reset stack pointer
       mov   es, dx         ; load pointer in ES:BX
       mov   bx, ax         ; copy double into save area (not shown)
       mov   word ptr MySaveArea, es:[bx]
       mov   word ptr MySaveArea+2, es:[bx+2]
       mov   word ptr MySaveArea+4, es:[bx+4]
       mov   word PTR MySaveArea+6, es:[bx+6]

 ^bFiles:^b  Library is CLIPPER.LIB, header file is Extend.h.
